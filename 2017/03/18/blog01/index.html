前端开发经典技巧总结

发表于 2017-03-18 | 分类于 js

前端开发经典技巧总结

  1. 首次给变量赋值切记使用var关键字

    变量没有赋值的话，默认会作为一个全局变量（一切无住的都是window的），要尽量避免全局变量.> 注意：在严格模式下（use strict）不声明将报错。

  2. 使用"==="代替"=="

    原因： ==和=！操作符存在隐式转换，导致”==”速度变慢，尽量能用全等就用全等。
    面试题：console.log([]==false);输出什么？输出true，两边隐式转换转换成了（0==0）;
    console.log([]===false)，输出false;

  3. undefined、null、0、false、NaN、“”（控制字符串）的逻辑结果均为false.

    可以利用这一特性减少代码如：if(data!=0){console.log(data)};可以直接写成if(data){console.log(data)};有些同学可能精简成data&&console.log(data);效果都一样，知识代码量减少一些，不过还是要注意一些代码风格，有些公司有要求的。。
    （好的代码像诗，坏的代码像屎）。

  4. 行尾使用分号

    实践中最好还是使用分号，忘记写也没事，大部分情况下javascript解释器会自动添加.
    影响：return {a=1} 如果不加”;”,js解析器会给你变成retturn; {a=1};
    这时候你只能哭了。。> 建议保持良好的编码习惯，可以使用webstorm这样强的带检错工具。
